# 接口联调异常场景测试用例

## 测试用例基本信息

| 属性 | 内容 |
|------|------|
| 用例集编号 | TC-INTF-ERR-001 |
| 所属测试项 | ATE 设备与 MES/EAP 接口联调 |
| 适用产品 | SoC 测试平台接口层 |
| 编写人 | 测试工程团队 |
| 版本 | V1.0 |

## 测试用例列表

### TC-001：MES 连接断开后自动重连验证
- **前置条件**：ATE 与 MES 已建立正常 TCP 连接，测试正在进行中
- **输入**：人工断开 MES 端网络连接，持续 30 秒后恢复
- **预期输出**：ATE 端检测到连接中断后缓存测试数据，网络恢复后自动重连并上传缓存数据，无数据丢失
- **判定**：Pass（重连时间 ≤ 10 秒，数据完整性 100%）
- **优先级**：P1

### TC-002：批次号格式错误拒绝测试
- **前置条件**：MES 下发批次信息时，批次号格式不符合规范（如含特殊字符）
- **输入**：接收格式错误的批次号报文
- **预期输出**：ATE 解析失败，返回错误码 ERR-4001（参数格式非法），不启动测试流程
- **判定**：Fail（正确触发拒绝逻辑）
- **优先级**：P1

### TC-003：EAP 指令超时处理
- **前置条件**：EAP 发送"开始测试"指令后，60 秒内无确认应答（模拟 ATE 假死）
- **输入**：EAP 超时重发指令（最多重试 3 次）
- **预期输出**：3 次重试后 EAP 记录异常日志，通知运维告警，不强制启动测试
- **判定**：Pass（超时处理流程正确执行）
- **优先级**：P1

### TC-004：并发批次指令冲突测试
- **前置条件**：ATE 正在执行批次 A 的测试任务
- **输入**：MES 同时下发批次 B 的启动指令
- **预期输出**：ATE 返回忙碌状态码 ERR-4002（设备占用），拒绝批次 B，批次 A 测试不受影响
- **判定**：Pass
- **优先级**：P2

### TC-005：测试数据上传失败重试机制
- **前置条件**：测试完成，ATE 向 MES 上传结果时，MES 服务端返回 503 错误
- **输入**：ATE 触发重试逻辑
- **预期输出**：每隔 5 秒重试一次，最多重试 5 次；超过重试上限后本地持久化存储，并触发人工介入告警
- **判定**：Pass（数据不丢失，重试日志完整）
- **优先级**：P2

## 回归测试要求

接口协议版本变更或 MES/EAP 软件升级后，TC-001 至 TC-003 为必须回归用例；TC-004 和 TC-005 在并发逻辑或重试机制涉及改动时纳入回归。
