# 软件版本升级验证测试用例

## 测试用例基本信息

| 属性 | 内容 |
|------|------|
| 用例集编号 | TC-SWU-VER-001 |
| 所属测试项 | ATE 测试系统软件版本升级验证 |
| 适用产品 | WPS-3000 / SoC 测试平台软件 |
| 编写人 | 测试工程团队 |
| 版本 | V1.3 |

## 测试用例列表

### TC-001：升级包完整性校验
- **前置条件**：已获取新版本升级包（.pkg 文件）
- **输入**：升级工具加载升级包，执行 SHA-256 哈希校验
- **预期输出**：校验值与发布清单中记录的哈希值完全一致，校验结果显示"通过"
- **判定**：Pass（不一致则拒绝升级并告警）
- **优先级**：P1

### TC-002：升级后版本号正确性验证
- **前置条件**：升级流程执行完成，系统重启
- **输入**：在系统信息页面查询当前版本号
- **预期输出**：显示版本号与升级包版本一致，旧版本不再显示
- **判定**：Pass
- **优先级**：P1

### TC-003：升级失败回滚验证
- **前置条件**：模拟升级过程中断电（在升级进度 50% 时强制断电）
- **输入**：重新上电后系统自动启动
- **预期输出**：系统检测到升级未完成，自动回滚至旧版本，提示"已恢复至 V{旧版本号}"，旧版本功能完全正常
- **判定**：Pass
- **优先级**：P1

### TC-004：升级后历史数据迁移验证
- **前置条件**：升级前记录系统中已有批次记录数量、测试程序列表
- **输入**：完成升级并重启
- **预期输出**：历史批次数据、测试程序配置和用户权限设置均保持不变，数量一致，内容无损
- **判定**：Pass
- **优先级**：P2

### TC-005：新旧版本接口兼容性验证
- **前置条件**：ATE 软件已升级至新版本，MES/EAP 暂未升级
- **输入**：MES 使用旧版协议向 ATE 发送批次下发指令
- **预期输出**：ATE 新版本正确解析旧版协议报文，测试正常启动，无兼容性报错
- **判定**：Pass
- **优先级**：P2

## 回归测试要求

每次正式版本发布前，TC-001 至 TC-003 为必须回归用例（属于升级安全门控项）；TC-004 在数据库结构有变更时必须纳入；TC-005 在接口协议涉及变更时纳入回归。
