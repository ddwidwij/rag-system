# 探针卡切换流程测试用例

## 测试用例基本信息

| 属性 | 内容 |
|------|------|
| 用例集编号 | TC-MECH-PCK-001 |
| 所属测试项 | 探针卡更换与切换操作规程 |
| 适用产品 | WPS-3000 探针台系统 |
| 编写人 | 测试工程团队 |
| 版本 | V1.1 |

## 测试用例列表

### TC-001：探针卡正常装载验证
- **前置条件**：测试机处于 Idle 状态，目标探针卡已通过外观检查，卡仓位空闲
- **输入**：操作员按规程将探针卡安装至卡仓，执行装载指令
- **预期输出**：系统识别探针卡 ID，OS 自检通过，显示"卡状态：就绪"
- **判定**：Pass
- **优先级**：P1

### TC-002：探针卡未锁定异常检测
- **前置条件**：探针卡未完全锁紧（模拟漏拧螺丝场景）
- **输入**：执行装载指令
- **预期输出**：机械传感器返回锁定失败信号，系统停止后续动作并弹出告警弹窗
- **判定**：Fail（触发 Alarm-PCK01 — 机械锁定异常）
- **优先级**：P1

### TC-003：错误型号探针卡拒绝安装
- **前置条件**：装入与当前测试程序不匹配型号的探针卡（如 WPS-2000 卡安装至 WPS-3000 台）
- **输入**：执行装载指令
- **预期输出**：系统读取探针卡 EEPROM 中的型号标识，比对失败后拒绝切换并提示型号不符
- **判定**：Fail（触发 Alarm-PCK02 — 型号不匹配）
- **优先级**：P1

### TC-004：探针卡切换后参数自动加载
- **前置条件**：前一探针卡已卸载，新探针卡装载成功
- **输入**：系统触发自动配置流程
- **预期输出**：针数、管脚映射表、接触力参数自动从数据库加载，无需人工干预
- **判定**：Pass
- **优先级**：P2

### TC-005：切换过程中急停恢复测试
- **前置条件**：探针卡正在执行切换动作中途
- **输入**：触发急停（E-Stop）按钮
- **预期输出**：运动立即停止，系统进入安全锁定状态；重新上电后可从断点继续或重新执行切换流程
- **判定**：Pass（急停响应时间 ≤ 200 ms）
- **优先级**：P2

## 回归测试要求

每次探针台固件版本升级或探针卡管理软件变更后，TC-001 至 TC-003 为必须回归用例；TC-004 和 TC-005 在涉及参数加载模块或运动控制模块变更时纳入回归。
