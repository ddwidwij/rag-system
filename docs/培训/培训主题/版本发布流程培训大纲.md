# 培训课程：版本发布流程培训

## 课程基本信息

| 属性 | 内容 |
|------|------|
| 培训主题编号 | TRN-SW-REL-001 |
| 培训主题 | 软件版本发布流程培训 |
| 适用岗位 | 软件工程师、测试工程师、产品经理 |
| 培训方式 | 线上理论（6h）+ 流程实践（4h）|
| 认证有效期 | 长期有效（流程变更时重修）|
| 培训负责人 | 软件质量部 |

## 培训目标

完成本培训后，学员应具备以下能力：
1. 理解 WPS-OS 软件版本号规则（Major.Minor.Patch-Build）
2. 按规范完成版本分支创建、构建和打包
3. 执行发布前测试检查单（Release Checklist）
4. 正确提交版本发布申请并完成审批流程
5. 完成版本发布后的客户通知与文档归档

## 培训大纲

### 模块一：版本管理基础（2h）
- 版本号规范（语义化版本 SemVer）
- Git 分支策略（main / release / hotfix / feature）
- 变更类型分类（功能新增 / 缺陷修复 / 安全补丁）
- 变更记录（Changelog）编写规范

### 模块二：版本构建与打包（2h）
- CI/CD 流水线概览
- 构建触发规则与构建产物说明
- 安装包签名与完整性校验
- 版本构建产物存档规范

### 模块三：发布前测试与审核（2h）
- Release Checklist 逐项说明
- 回归测试范围与通过标准
- 安全扫描与漏洞评估要求
- 版本评审会（Release Review）流程

### 模块四：发布与通知（2h）
- 发布申请提交（含版本说明、影响范围、回滚方案）
- 版本发布操作步骤（内部升级 + 客户推送）
- 客户发布通知邮件模板
- 发布后监控与回滚触发条件

### 模块五：流程实践（4h）
- 演练：按流程完成一次模拟 Patch 版本发布
- 检查 Checklist 完整性，提交模拟审批
- 撰写 Release Notes 并归档

## 考核标准

| 考核项 | 权重 | 合格线 |
|-------|------|-------|
| 流程知识测试 | 40% | ≥ 80 分 |
| 模拟发布演练 | 60% | Checklist 完成率 ≥ 95% |

综合得分 ≥ 80 分，获得版本发布操作权限。
