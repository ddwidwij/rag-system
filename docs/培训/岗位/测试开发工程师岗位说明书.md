# 岗位说明书：测试开发工程师

## 岗位基本信息

| 属性 | 内容 |
|------|------|
| 岗位编号 | POS-RD-TDE-001 |
| 岗位名称 | 测试开发工程师（Test Development Engineer） |
| 所属部门 | 研发部 / 测试部 |
| 直属上级 | 测试开发组长 |
| 岗位级别 | 工程师级（P3~P5） |
| 工作地点 | 公司研发中心 |

## 岗位职责

1. **测试程序开发**：基于 ATE 平台（WPS-3000 / SoC 测试机）开发芯片 CP/FT 测试程序
2. **测试方案设计**：根据芯片规格书制定测试策略，定义测试项、判定门限和 Bin 分配
3. **调试与优化**：在硅片和封装样品上调试测试程序，优化 ATT（平均测试时间）和良率相关性
4. **硬件协作**：与硬件工程师协作完成探针卡、DIB 板（Device Interface Board）设计评审
5. **问题分析**：对测试异常（良率波动、Bin 异常）进行根因分析，输出分析报告
6. **文档编写**：编写测试程序说明文档、测试方案书和 Tester Setup 规范
7. **技术支持**：支持量产阶段测试程序维护，响应量产异常并快速迭代修复

## 任职要求

### 学历与专业
- 本科及以上，微电子、集成电路、电子工程或相关专业

### 工作经验
- P3：1~3 年，具备独立开发单模块测试程序的能力
- P4：3~6 年，能独立完成复杂 SoC 芯片全套测试程序开发
- P5：6 年以上，具备测试平台架构设计能力，可带领团队攻关

### 技能要求
- 熟练掌握 C/Python 编程，具备 ATE 测试程序开发经验
- 熟悉半导体测试原理（DC 参数、功能测试、RF 测试）
- 能阅读芯片规格书（Datasheet）并从中提取测试需求
- 了解 ATPG、Scan 测试、DFT 基础知识

## 绩效考核维度

| 考核项 | 权重 | 目标值 |
|-------|------|-------|
| 程序开发按时交付率 | 35% | ≥ 90% |
| 测试程序缺陷逃逸率 | 30% | ≤ 2 项/季度 |
| ATT 优化达成率 | 20% | 较初版降低 ≥ 15% |
| 文档完整率 | 15% | 100%（交付时必须附文档） |

## 职业发展路径

测试开发工程师 → 高级测试开发工程师 → 测试开发专家 / 测试架构师
