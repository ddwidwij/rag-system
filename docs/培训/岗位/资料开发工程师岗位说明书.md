# 岗位说明书：资料开发工程师

## 岗位基本信息

| 属性 | 内容 |
|------|------|
| 岗位编号 | POS-TD-DW-001 |
| 岗位名称 | 资料开发工程师（Technical Writer / Documentation Engineer） |
| 所属部门 | 产品管理部 / 研发部 |
| 直属上级 | 产品文档负责人 |
| 岗位级别 | 工程师级（P2~P4） |
| 工作地点 | 公司本部 |

## 岗位职责

1. **技术文档编写**：编写产品用户手册、安装手册、维修手册、快速参考指南等
2. **规格书整理**：配合研发整理芯片或设备规格书（Datasheet），确保准确性和完整性
3. **培训材料制作**：根据培训大纲制作 PPT、操作视频脚本及配套练习题
4. **文档版本管理**：维护文档版本台账，跟踪变更历史，确保客户使用最新受控版
5. **翻译与本地化**：将核心技术文档翻译为英文或其他目标语言版本
6. **知识库建设**：将散落在邮件、会议纪要中的技术知识结构化，录入企业知识管理平台
7. **文档评审协调**：组织文档内部评审，收集研发、测试、服务部门的反馈并整合修订

## 任职要求

### 学历与专业
- 本科及以上，电子工程、技术传播、中文或相关专业；理工科背景优先

### 工作经验
- P2：0~2 年，具备良好的技术文档写作基础，能在指导下完成模块文档
- P3：3~5 年，能独立负责完整产品文档体系，具备跨部门协调能力
- P4：5 年以上，能制定文档规范和模板体系，主导知识库架构设计

### 技能要求
- 出色的中文技术写作能力，英文读写流利
- 熟悉 Markdown、Confluence、MadCap Flare 等文档工具
- 具备基础技术理解能力，能读懂电路框图、流程图和接口时序图
- 熟悉版本控制工具（Git）优先

## 绩效考核维度

| 考核项 | 权重 | 目标值 |
|-------|------|-------|
| 文档按时交付率 | 35% | ≥ 95% |
| 文档评审一次通过率 | 30% | ≥ 85% |
| 知识库录入条目数 | 20% | ≥ 8 条/季度 |
| 客户文档投诉次数 | 15% | 0 次/季度 |

## 职业发展路径

资料开发工程师 → 高级资料开发工程师 → 技术文档经理 / 知识管理专家
