
## 文档解析

这份需求文档描述的是一个**面向半导体检测设备企业的内部知识底座 + AI助手平台**，核心三大目标：

| 目标 | 定义 | 核心路径 |
|------|------|----------|
| 目标1 | 低级文本问题自动检查率 80% | 规则校验 + AI语义检查 |
| 目标2 | 技术参数准确性比对覆盖率 50% | 知识库/RAG + 结构化比对 |
| 目标3 | 产品文档自动生成率 30% | AI草稿生成/改写/翻译 |

---

## 初步简单实现方案

基于现有 RAG 系统基础（ChromaDB + SentenceTransformer + DeepSeek），按**三阶段**推进：

### 阶段一：快速跑通核心闭环（4-6周）

**重点聚焦目标1 + 目标2 基础版**

```
现有 RAG 系统
    ↓ 扩展
┌─────────────────────────────────────────┐
│  文档接入层                              │
│  Markdown/Word → 解析 → ChromaDB        │
├─────────────────────────────────────────┤
│  规则校验层（Python脚本，确定性）         │
│  · 错别字检查（pypinyin + 词典）         │
│  · 编号/格式一致性（正则）               │
│  · 模板缺项检查（Schematron / 自定义）   │
├─────────────────────────────────────────┤
│  AI语义层（现有RAG）                     │
│  · 歧义/重复/语气问题（DeepSeek）        │
│  · 技术参数问答（RAG检索 + 比对）        │
├─────────────────────────────────────────┤
│  输出层                                  │
│  · Web前端问答页（现有 server.py 扩展）  │
│  · HTML检查报告                          │
└─────────────────────────────────────────┘
```

**具体任务清单：**

1. **扩展文档解析**：用 `unstructured` 或 `python-docx` 支持 Word/DITA XML 格式接入
2. **实现规则检查器**：Python 脚本，检查错别字（结合 `pycorrector`）、格式规范（正则）、模板缺项
3. **参数比对模块**：维护术语表/参数表（Excel/JSON），从文档中抽取关键参数后做结构化比对
4. **扩展 Web 前端**：在现有 server.py 基础上增加"文档检查"入口，返回问题列表

**技术栈（最小依赖）：**
```
pycorrector          # 中文纠错
unstructured         # 多格式文档解析
python-docx          # Word 解析
lxml                 # DITA/XML 解析
fastapi + 现有server # Web服务（已有基础）
```

---

### 阶段二：技术比对 + 知识治理（2-3个月）

- 引入**元数据字段**（产品线、版本、部门、保密等级）
- 构建**术语表/参数表知识库**，实现"文档抽取参数 → 权威源比对"
- 接入 DITA 解析（`lxml`），支持 conref/xref 异常检测
- 增加**差异报告**输出（HTML/JSON）

---

### 阶段三：文档生成 + 平台化（3-6个月）

- AI 草稿生成（基于模板 + DeepSeek/本地模型）
- 接入权限控制（部门/项目级隔离）
- 替换向量库为 **Milvus** 或 **OpenSearch**（支持权限过滤）
- 本地部署模型（Qwen/DeepSeek via vLLM）

---

## 最小可行版（MVP）建议

**优先只做这3件事打通闭环：**

1. **规则检查器** → 输入文档，输出问题列表（1-2周）
2. **RAG问答** → 基于现有系统，接入产品规格书、FAQ（已有基础，扩充文档即可）
3. **Web页面** → 问答+检查报告双入口（现有 server.py 扩展）
