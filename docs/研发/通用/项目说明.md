# 文档中心工作区说明

这个 README 说明当前工作区根目录下各文件和目录的作用，并标注哪些内容是当前主项目在使用的，哪些更像历史产物或独立预览文件。

## 当前目录结构

### 根目录文件

| 路径 | 作用 | 当前是否在主项目中使用 |
|---|---|---|
| `.DS_Store` | macOS 自动生成的目录元数据文件。 | 否 |
| `login.html` | 根目录下的独立静态登录页原型，包含完整 HTML/CSS/JS，可直接浏览器打开预览。 | 否。未被 `tdc-doc-center` 主项目引用。 |
| `Login.vue` | 根目录下的单文件 Vue 登录页原型，是把静态登录页改写成 Vue SFC 的独立版本。 | 否。当前没有任何项目引用这个文件。 |
| `README.md` | 当前工作区说明文档。 | 是 |
| `文档中心方案.docx` | 文档中心相关方案说明文档，属于需求/方案资料。 | 否，非运行时文件 |

### 根目录文件夹

| 路径 | 作用 | 当前是否在主项目中使用 |
|---|---|---|
| `OUT/` | 一套 DITA/手册导出结果，含 `index.html`、`commonltr.css`、`commonrtl.css` 以及各子目录手册页面。更像原始/外部导出产物。 | 否，不直接被 `tdc-doc-center` 主项目运行时引用 |

| `vue-login-preview/` | 一个独立的 Vue 预览小项目，用于单独运行和查看 Vue 版登录页。 | 否，不是主项目的一部分 |

## `login.html` 和 `Login.vue` 哪个没用

结论：**根目录下这两个文件当前都没有被主项目使用。**

### `./login.html`

- 是一个独立静态原型页。
- 当前 `tdc-doc-center` React 主项目并不引用它。
- 主项目里真正相关的静态登录页是：
  - `tdc-doc-center/apps/web/public/login.html`

### `./Login.vue`

- 是一个独立的 Vue 单文件组件原型。
- 当前没有任何地方 import 它。
- `vue-login-preview` 这个独立 Vue 项目实际使用的是：
  - `vue-login-preview/src/views/LoginView.vue`

### 实际判断

如果只看“当前主项目 `tdc-doc-center` 是否使用”：

- `./login.html`：没用
- `./Login.vue`：没用

如果只看“当前整个工作区是否还有参考价值”：

- 两者都属于历史原型/备份文件，可保留作参考
- 但都不是当前运行链路的一部分

## 当前真正运行链路

如果你现在启动和维护的是文档中心系统，真正有效的是：

1. `tdc-doc-center/apps/web/src/` 前端 React 页面
2. `tdc-doc-center/apps/api/src/` 后端接口
3. `tdc-doc-center/manuals/` 手册内容
4. `tdc-doc-center/apps/web/public/` 静态公共资源

根目录下的 `login.html`、`Login.vue`、`vue-login-preview/` 更像历史演示或独立预览材料，不属于当前生产运行路径。

## 整理建议

为了方便后续维护，可以把当前工作区内容按下面 4 类理解。

### 1. 运行必需

这些内容是当前主项目真正依赖的，日常开发和运行不要删：

- `tdc-doc-center/`
- `tdc-doc-center/apps/web/src/`
- `tdc-doc-center/apps/api/src/`
- `tdc-doc-center/manuals/`
- `tdc-doc-center/apps/web/public/`
- `tdc-doc-center/package.json`
- `tdc-doc-center/docker-compose.yml`

### 2. 可选预览

这些内容不是主项目的一部分，但如果你还需要单独演示某个版本的登录页，可以保留：

- `vue-login-preview/`

适用场景：
- 想单独跑一个 Vue 版登录页预览
- 想保留 Vue 技术路线参考

### 3. 历史原型 / 参考资料

这些文件当前不参与运行，但可能有参考价值：

- `login.html`
- `Login.vue`
- `OUT/`
- `文档中心方案.docx`

适用场景：
- 回看早期登录页原型
- 对比 DITA 导出原始样式
- 查方案说明

### 4. 可清理候选

如果你的目标是“只保留当前主项目和必要资料”，以下内容都可以作为清理候选。

#### 可以优先清理

- `./login.html`
- `./Login.vue`

原因：
- 当前都没有被引用
- 功能上与主项目运行链路无关

#### 视情况清理

- `vue-login-preview/`
- `OUT/`

原因：
- 它们有参考价值，但不是主项目必需
- 如果你不再需要 Vue 登录预览，`vue-login-preview/` 可以清理
- 如果 `OUT/` 只是历史导出样本，而你真正使用的是 `tdc-doc-center/manuals/`，也可以考虑归档到别处

#### 不建议直接清理

- `tdc-doc-center/`

原因：
- 这是当前真正的业务项目

## 最简保留方案

如果你以后想把工作区整理成“最小可维护版本”，建议至少保留：

- `tdc-doc-center/`
- `文档中心方案.docx`（如果你还需要方案文档）
- `README.md`

其他如 `login.html`、`Login.vue`、`vue-login-preview/`、`OUT/` 可以根据你是否还需要参考再决定是否归档。
