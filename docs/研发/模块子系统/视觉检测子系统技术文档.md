# 视觉检测子系统技术文档

## 子系统概述

视觉检测子系统（Vision Inspection Module，简称 VIM）是 WPS 系列测试机的核心感知与定位模块，承担晶圆对准、针尖识别、Mark 定位、图像判读和过程监控职责。该子系统直接影响设备的首片对准时间、重复定位精度和异常恢复效率，因此在整机架构中属于高优先级模块。对于晶圆 CP、FT 抽测、Bumped Wafer 对准和视觉辅助校准场景，视觉子系统不仅要保证识别精度，还要在不同工艺表面、不同反光条件和不同光照环境下维持稳定表现。

## 模块架构

```
视觉检测子系统
├── 上方相机（Top Camera）       — 探针卡针尖与参考标记识别
├── 下方相机（Bottom Camera）    — 晶圆 Fiducial / Die Mark 识别
├── 光源控制器（Light Controller）— 同轴光、环形光、斜侧光切换
├── 图像处理单元（IPU）          — 图像采集、滤波、模板匹配、坐标解算
└── 标定与配方管理模块            — 标定文件、Recipe、阈值参数管理
```

系统通过运动控制总线接收当前位置，通过 IPC 与主控软件交换配方参数和识别结果。识别结果输出包括晶圆坐标偏差、旋转角、针尖补偿量和图像质量评分，用于驱动后续运动补偿与测试流程放行。

## 核心功能

### 1. 自动对准流程

视觉子系统在换片完成后自动执行三段式对准。第一阶段识别 Notch/Flat 方向，完成粗定位；第二阶段读取 Fiducial Mark 或参考 Die 特征，进行亚像素级精定位；第三阶段调用探针卡针尖识别算法，对针尖阵列位置与晶圆参考坐标做差值补偿。标准条件下从取图到输出补偿量的总耗时控制在 1.2 秒以内。

### 2. 晶圆 Map 与异常标记识别

系统支持读取 XML、CSV、SECS/GEM 转换格式的 Wafer Map 文件，将 Ink Dot、Skip Die、Fail Die、Sampling Die 等标记同步到视觉配方中。视觉算法在搜索窗口内优先避开已标记区域，减少误识别和无效测试。对于局部污染、边缘崩裂或表面反光异常的晶圆，可通过配方切换识别模式与阈值。

### 3. 针尖检测与漂移补偿

上方相机在设备开机、自检和周期性复核阶段执行针尖阵列扫描，统计针尖形貌、间距偏差和热漂移量。当系统检测到针尖坐标超出设定容差时，会向主控软件返回告警，并阻止自动进入批量测试状态。该机制可有效降低因针尖磨损、板卡热胀冷缩和探针卡装夹偏差带来的误判风险。

## 关键参数

| 参数 | 规格 |
|------|------|
| 相机分辨率 | 2048 × 2048 像素 |
| 下相机视野范围 | 8 mm × 8 mm |
| 上相机视野范围 | 5 mm × 5 mm |
| 单次取图至解算时间 | ≤ 1.2 秒 |
| 重复定位精度 | ±0.3 µm |
| 支持晶圆类型 | 200 mm / 300 mm / Bumped Wafer |

## 接口与维护要点

视觉子系统与运动控制子系统通过坐标补偿接口交互，与配方管理模块通过文件和数据库双通道同步阈值参数，与报警系统共享图像质量评分和识别失败原因码。维护时应重点关注以下事项：

- 每月清洁镜头、保护玻璃和光源窗口，避免油污与灰尘影响对比度。
- 每季度执行一次完整标定，更新 `/config/vision/` 下的相机矩阵和畸变参数。
- 当客户切换新工艺晶圆或新表面材料时，需先在工程模式下验证光源与阈值，不建议直接沿用历史配方。
- 光源灯珠累计工作 5000 小时后应做亮度均匀性检查，必要时整体更换。
