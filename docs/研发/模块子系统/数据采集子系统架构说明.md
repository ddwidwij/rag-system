# 数据采集子系统架构说明

## 子系统概述

数据采集子系统（Data Acquisition Module，简称 DAM）是 WPS 系列测试机的测量结果汇聚与持久化核心，负责将测试头板卡、量测板卡、温控单元、运动控制器和外部工艺接口产生的原始采样数据统一接入、缓存、标记并输出至上层软件和客户数据系统。该子系统既要满足高频采样（>10 kSa/s）的低延迟要求，又要保证每个量测点都能关联到 Lot、Wafer、Die、Recipe、软件版本和板卡状态，实现全链路可追溯。对于量产测试、良率分析和在线 SPC 场景，数据采集子系统的可靠性与完整性是整机数据质量的最终保障。

## 模块架构

```
数据采集子系统
├── 板卡驱动接入层（Driver Abstraction Layer）  — 统一封装各板卡采样接口
├── 采样调度与高速缓存层                        — 触发采样、写入内存队列、异步刷盘
├── 批次上下文管理与结果标记层                  — 绑定 Lot/Wafer/Die/Recipe 上下文
├── 格式输出适配层                              — STDF / CSV / 内部二进制多格式导出
└── 异常抓取与日志回放模块                      — 保留故障前后关键窗口的原始快照
```

驱动接入层对测试头板卡、温控单元和运动控制器提供统一采样 API；调度层根据 Recipe 定义的测试序列触发采样并写入高速环形缓存；标记层将结果与设备上下文绑定；输出层按客户配置导出 STDF、CSV 或内部分析格式，并通过异步写盘避免 IO 抖动影响测试节拍。

## 核心功能

### 1. 高一致性采样调度

同一测试流程中的量测值、告警码、时间戳和环境参数必须在同一上下文中落盘，防止结果与条件脱节。系统通过批次上下文 ID 和全局测试序列号实现跨模块关联，即使在多板卡并行采样场景下也能保证数据顺序完整性。

### 2. 性能与容错保障

高频采样阶段优先写入内存队列和本地高速 SSD 缓存，再通过异步线程批量刷盘。当磁盘抖动或网络中断时，系统允许短时缓冲，但不允许静默丢失任何采样点。当缓存占用超过阈值时，系统主动告警并暂停新任务派发，保护已积累数据。

### 3. 可追溯多格式输出

所有导出结果必须包含最小追溯字段：设备编号、软件版本、配方版本、时间戳、操作员 ID 和板卡序列号。STDF 格式按 SEMI E10 标准输出，支持 PRR/PTR/FTR 等标准记录类型；CSV 格式支持客户自定义列映射，映射规则受控管理，不允许现场临时改列名。

## 关键参数

| 参数 | 规格 |
|------|------|
| 最大采样频率 | 10 kSa/s（单通道） |
| 采样触发延迟 | ≤ 0.5 ms |
| 内存缓存深度 | 128 MB 环形缓存 |
| 支持输出格式 | STDF v4 / CSV / 内部二进制 |
| 单片数据落盘时间 | ≤ 200 ms |
| 最大并行板卡数 | 16 块 |

## 接口与维护要点

数据采集子系统通过驱动接口读取测试头板卡原始量测，通过配方接口从主控获取采样触发配置，通过 STDF/CSV 导出接口向 MES 和数据湖推送批次报表。维护时应重点关注以下事项：

- 版本升级前核查字段映射表与输出格式兼容性，防止新版本字段改动破坏客户报表。
- 当客户新增报表字段时，必须同步更新数据字典并补充接口测试用例，不允许只在前端配置。
- 本地缓存目录需定期巡检容量与文件完整性，防止异常日志将正常数据覆盖。
- 现场排障时优先保留原始采样快照，再做二次分析，避免丢失问题复现证据。
- 磁盘空间低于 20% 时应提前告警，避免因空间不足导致数据丢失。
