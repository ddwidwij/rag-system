# 温控子系统稳定性设计说明

## 子系统概述

温控子系统（Temperature Control Module，简称 TCM）为 WPS 系列晶圆测试设备提供稳定、可重复的热环境，覆盖常温测试（25°C）、低温筛选（-40°C）、高温老化（+175°C）与热循环验证等全场景工况。系统由 Chuck 本体、加热/制冷执行机构、循环介质控制单元、温度采样链路和闭环控制算法组成。对功率器件、车规芯片和高精度模拟芯片而言，温度波动直接影响漏电流、阈值电压、时序参数和接触稳定性，因此温控系统的设计重心不是极限升降温速度，而是目标温度的稳态保持能力和批次间一致性。

## 模块架构

```
温控子系统
├── Chuck 本体与热接触层                    — 高导热结构、分区控温、表面吸附
├── 加热执行机构（Heater Assembly）         — 分区加热膜、功率驱动与过温保护
├── 制冷执行机构（Cooling Assembly）        — 液冷/冷媒循环、流量与压力监测
├── 温度采样链路（Temp Sensor Chain）       — 主/冗余传感器双路采集、偏差监控
└── 闭环控制器（PID + Feedforward Controller）— 升温、稳态、降温三段参数切换
```

Chuck 本体采用高导热均温板结构，通过分区加热膜独立控温降低中心与边缘温差；制冷机构根据目标温度选择液冷或冷媒辅助，并持续监测流量和压力；温度采样链路配置主传感器与冗余传感器双路，偏差超限时禁止进入量产状态；控制器根据当前阶段（升温/稳态/降温）自动切换 PID 参数组。

## 核心功能

### 1. 热容量与结构均匀性控制

Chuck 采用高导热铝合金或氮化铝基板，结合分区加热设计，针对 200 mm 和 300 mm 晶圆分别优化热传导路径。通过更换热接触层和调整配方参数，适应不同工艺晶圆（裸硅、Bumped Wafer、Thin Die）的热特性差异，避免局部热点影响接触稳定性和测试一致性。

### 2. 多阶段闭环控制算法

控制器采用 PID 主控加前馈补偿策略：升温阶段优先保证响应速度，接近目标温度前自动切换稳态参数组以减小超调；长时间恒温阶段根据温漂趋势做微量修正，避免频繁大幅动作干扰测试稳定性；降温阶段开启主动冷却并监控降速曲线，防止因过快冷却导致 Chuck 表面结露。

### 3. 多级保护与异常响应

系统内置四级热保护：当温度超过上限阈值时立即切断加热输出；当温度变化速率异常时执行受控降功率；当主/冗余传感器偏差超出容差时禁止进入量产测试；当冷却流量低于下限时触发告警并自动降温。对于可疑故障，系统保存故障前后 60 秒的温度曲线快照，供后续分析。

## 关键参数

| 指标项 | 设计值 |
|------|------|
| 控温范围 | -40°C ~ +175°C |
| 稳态温度波动 | ±0.2°C |
| Chuck 中心与边缘温差 | ≤ 0.8°C（300 mm 晶圆） |
| 25°C → 150°C 升温时间 | ≤ 8 分钟 |
| 150°C 恒温 30 分钟漂移 | ≤ ±0.3°C |
| 主/冗余传感器允许偏差 | ≤ 0.5°C |

## 接口与维护要点

温控子系统通过 Chuck 坐标接口与运动控制子系统联动（禁止在未到温时执行接触动作），通过配方接口接收主控软件的目标温度和升降温曲线，通过冷却系统监控接口与设备安全 PLC 共享冷却状态信号。维护时应重点关注以下事项：

- 每月检查冷却回路流量、压力和管路泄漏情况，确认滤芯状态符合更换周期。
- 每季度校准主/冗余温度传感器并留存校准记录，传感器偏差趋势异常时应提前更换。
- 客户更换 Chuck 表面涂层、吸附方式或晶圆尺寸后，必须重新验证温度均匀性，不允许直接沿用原配方。
- 发生超温故障后，需同时排查加热控制、冷却链路、算法参数和负载条件是否联动变化，不允许仅复位后重启。
- 低温测试结束后应按规程执行回温除露，防止冷凝水损坏 Chuck 表面和接触结构。
