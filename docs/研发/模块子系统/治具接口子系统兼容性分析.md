# 治具接口子系统兼容性分析

## 子系统概述

治具接口子系统（Fixture Interface Module，简称 FIM）是 WPS 标准测试平台向项目化客制交付延伸的关键边界模块，负责连接整机测试资源与客户定制治具、探针卡接口板、信号转接板和外部辅助单元。该子系统的核心挑战在于：不同针卡结构、不同 DUT 引脚定义、不同信号电平和不同机械锁附方式，必须在统一平台上稳定共存，且换型操作不引入额外风险。兼容性分析需同时覆盖机械、电气、软件三个层级，不能仅做接线正确性检查。

## 模块架构

```
治具接口子系统
├── 机械接口层（Mechanical Interface）       — 安装基准、锁紧机构、平面度管控
├── 电气接口层（Electrical Interface）       — 引脚映射、信号电平、保护回路
├── 信号转接板（Signal Relay Board）         — 标准平台信号到 DUT 引脚的适配
├── 配方与通道映射管理（Recipe & Mapping）   — 治具编码与限值/保护策略绑定
└── 兼容性验证与记录（Verification Log）     — 新治具导入、首件验证、清单维护
```

机械接口层确保治具安装时平面度和基准一致；电气接口层负责信号完整性与保护合规；信号转接板承接标准平台到 DUT 的映射；配方管理层将治具编码与受控 Recipe 绑定，避免人工切换参数引入风险；验证记录层保存历次导入结论和兼容性清单。

## 核心功能

### 1. 机械兼容验证

新治具导入时，需确认安装基准尺寸、锁紧方式、平面度公差和更换操作空间。机械基准不统一会在接触阶段导致偏移、受力不均和探针损伤，即使电气映射正确也无法正常测试。系统要求所有新治具通过机械评审后才能进入电气联调阶段。

### 2. 电气兼容分析

接口定义需全面覆盖数字、模拟、高压、接地和屏蔽信号通道。重点验证最大电压/电流、阻抗匹配、串扰敏感度及保护回路规格是否与平台标准一致。对于客户自定义接口板，必须逐引脚核查映射关系、保护器件规格和回路地线拓扑，不允许使用未评审的临时接线方案。

### 3. 软件与配方兼容

每种治具必须在系统中建立独立的受控 Recipe，包含通道映射、量测限值、高压保护阈值和时序参数。多治具共台时，切换操作必须触发 Recipe 强制选择确认，不允许依赖人工记忆操作。主控软件需识别治具编码并自动加载对应配方，错误配方加载时应拒绝进入测试。

## 关键参数

| 兼容性维度 | 关键控制项 | 验证方式 |
|------|------|------|
| 机械接口 | 基准尺寸、平面度、锁紧力矩 | 尺寸测量 + 安装复核表 |
| 电气接口 | 引脚映射、耐压等级、绝缘电阻 | 交叉核对 + 耐压测试 |
| 信号质量 | 串扰、阻抗、电平范围 | 示波器抓波 + 限值核对 |
| 软件配方 | 通道映射、限值、保护参数 | 黄金样品验证 + 抽测 |

## 接口与维护要点

治具接口子系统与测试头板卡子系统共享电气互连基准，与主控软件的配方管理模块联动，与安全互锁系统共享高压保护回路状态。维护时应重点关注以下事项：

- 每次换型前必须确认治具编码与配方版本一致，禁止在未更新记录的情况下跨治具复用配方。
- 高压接口绝缘清洁应列入月度 PM 计划，防止油污导致绝缘下降和高压漏电。
- 兼容性清单需包含治具编号、适用机型、配方版本和验证日期，由研发和品质双签管控。
- 客户现场新增接口板或探针卡前，必须先完成机械、电气评审，不允许未经评审直接上机测试。
- 每次治具更换后应执行连通性和基准校验，发现异常立即上报，不得强行继续生产。
