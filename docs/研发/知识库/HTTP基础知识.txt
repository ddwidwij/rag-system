HTTP 协议基础知识

HTTP（超文本传输协议）是互联网上应用最广泛的网络协议，用于客户端与服务器之间的数据传输。

HTTP 请求方法

GET 方法用于从服务器获取资源，不会修改服务器上的数据，属于幂等且安全的操作。
POST 方法用于向服务器提交数据，常用于创建新资源或提交表单数据。
PUT 方法用于更新服务器上已存在的资源，需要提供完整的资源内容，属于幂等操作。
DELETE 方法用于删除服务器上的指定资源，执行后资源将被移除。
PATCH 方法用于对资源进行局部更新，只需提供需要修改的字段。
HEAD 方法与 GET 类似，但服务器只返回响应头，不返回响应体，常用于检查资源是否存在。
OPTIONS 方法用于获取目标资源所支持的通信选项，常用于 CORS 预检请求。

HTTP 状态码

状态码是服务器对客户端请求的响应结果的标准化数字编码。

1xx 信息性状态码：表示请求已被接收，需要继续处理。
100 Continue 表示客户端可以继续发送请求的剩余部分。
101 Switching Protocols 表示服务器同意将协议切换到客户端请求的协议。

2xx 成功状态码：表示请求已成功被服务器接收、理解并处理。
200 OK 是最常见的成功状态码，表示请求成功。
201 Created 表示请求成功并创建了新资源，通常用于 POST 请求。
204 No Content 表示请求成功但响应体为空，通常用于 DELETE 请求。

3xx 重定向状态码：表示需要客户端采取进一步操作来完成请求。
301 Moved Permanently 表示资源已永久迁移到新 URL。
302 Found 表示资源临时位于不同 URL。
304 Not Modified 表示资源未发生变化，客户端可使用缓存版本。

4xx 客户端错误状态码：表示客户端发送的请求有错误。
400 Bad Request 表示请求格式错误或参数不合法。
401 Unauthorized 表示请求未经认证，需要提供有效的身份凭证。
403 Forbidden 表示服务器拒绝执行该请求，即使已认证也没有权限。
404 Not Found 表示请求的资源不存在于服务器上。
405 Method Not Allowed 表示请求的 HTTP 方法不被目标资源支持。
429 Too Many Requests 表示客户端在单位时间内发送了过多请求，触发了限流。

5xx 服务器错误状态码：表示服务器在处理请求时发生错误。
500 Internal Server Error 表示服务器内部发生未知错误。
502 Bad Gateway 表示网关或代理服务器从上游服务器收到无效响应。
503 Service Unavailable 表示服务器暂时无法处理请求，通常因为过载或维护。
504 Gateway Timeout 表示网关或代理服务器等待上游响应超时。

HTTP 请求头与响应头

Content-Type 头用于指定请求体或响应体的媒体类型，例如 application/json 表示 JSON 格式数据。
Authorization 头用于携带认证信息，常见格式包括 Bearer Token 和 Basic 认证。
Accept 头用于告知服务器客户端可以处理的响应内容类型。
Cache-Control 头用于控制缓存行为，例如 no-cache 表示每次请求都需要向服务器验证。
